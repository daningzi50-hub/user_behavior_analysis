# 淘宝电商用户增长诊断与运营策略优化  
**——基于亿级行为数据的RFM+漏斗分析**

### 项目背景
针对淘宝平台用户转化效率低、留存不足的问题，本项目使用阿里云天池淘宝亿级用户行为公开数据集，抽样10万用户全量行为数据（约1000万条记录），从用户获取、留存、转化、价值分层和商品热度等多维度进行诊断分析，并提出可落地的运营优化建议，助力平台提升GMV和用户粘性。

**数据来源**：https://tianchi.aliyun.com/dataset/649  
**技术栈**：Python（数据抽样）、MySQL（清洗与分析）、Tableau（可视化仪表盘）

### 数据预处理
- 使用Python随机抽取10万用户，保留其完整行为日志。
- MySQL完成数据清洗：去除重复记录（7条）、异常时间戳（5303条）、负时间戳（93条），最终保留约1008万条有效记录。
- 说明：因采集周期较短，部分用户存在购买记录但无浏览记录；数据无注册时间、无消费金额，采用合理代理指标进行分析。

### 核心分析与业务洞察

**1. 用户转化漏斗分析**  
构建“浏览 → 收藏/加购 → 购买”核心漏斗路径。  
- 收藏加购率约9%，收藏/加购环节转化率约5%，整体转化率仅0.5%（购买率2.2%）。  
**核心洞察**：用户兴趣被激发后，在决策环节流失严重，是转化瓶颈。  
**运营建议**：在收藏/加购页面增加限时优惠、满减凑单、AI智能搭配推荐等干预措施，预计可将该环节转化率提升至8-12%，整体转化率有望提升至0.8%-1.0%以上。

**2. 用户留存与时间序列分析**  
因数据无注册信息，以用户首次出现次日重复活跃比例计算次日留存率约80%。  
结合时间序列分析发现：用户活跃时间高度集中在晚上23点，存在明显峰值。  
**核心洞察**：留存数字虽较高，但样本为已产生行为的用户，可能高估真实新用户留存；23点为活跃高峰。  
**运营建议**：针对首次行为用户，在次日20-23点推送个性化新人任务、优惠券或站内信，提升二次活跃；同时建议平台完善注册环节数据采集，提升留存分析准确性。

**3. RFM用户分层（调整版）**  
因缺乏消费金额数据，调整为RF模型（Recency最近活跃 + Frequency活跃频率），将用户分为**价值 / 保持 / 发展 / 挽留**四类。  
**核心洞察**：价值用户占比虽低，但贡献大量浏览和加购行为；挽留用户占比高但近期无活跃，流失风险大。  
**运营建议**：  
- 价值用户：实施VIP权益、精准营销和专属推荐；  
- 挽留用户：推送召回优惠券和站内信，目标将10%左右的挽留用户转化回“保持”层；  
- 发展/保持用户：通过差异化内容推送，提升活跃频率。

**4. 商品热度与品类分析**  
提取商品和品类Top10热度，通过透视表分析各品类浏览量与转化表现。  
**核心洞察**：部分高热度品类转化表现突出，但整体转化率仍有提升空间。  
**运营建议**：加大高热度品类曝光力度，结合跨品类搭配营销，提升整体GMV贡献。

### 项目总结与收获
本项目从海量原始数据出发，完成了“数据清洗 → 多维度分析 → 可视化呈现 → 业务建议”的完整闭环。  
通过将技术指标转化为具体运营动作，我初步掌握了数据驱动业务决策的思维方式。未来可进一步补充注册/金额数据，或引入A/B测试验证优化效果。
